Context: GreenPoll is a software-only pipeline for greenhouse pollination robotics. CrossPoll is the core research method — cross-crop transfer learning with YOLOv8n to detect flowers under limited label budgets.
🧩

Problem Terms

Vocabulary
Flower detection

Locate flowers in images via bounding boxes with a class label.

Object detection

Predict class + bounding-box location simultaneously in one forward pass.

Class label

Crop-specific flower tag, e.g. tomato_flower, kiwi_flower.

Domain shift

Crop type, lighting, or camera conditions differ between training and test sets.

Cross-crop transfer

Learn from source crops (tomato, strawberry, apple); adapt to unseen crops (kiwi, pepper).

Label budget

Number of annotated examples available: N = 25, 50, 100, 200, 500.

⚙️

Method Terms

Vocabulary
TermPractical meaning in this project
Transfer learningStart from pretrained weights; adapt the whole or part of the network.
PretrainingLearn generic / source-crop representations first before target adaptation.
Fine-tuningContinue training on target data with a lower learning rate after pretraining.
Few-shot adaptationFine-tune at low label budgets (N = 25 → 500) to measure sample efficiency.
Sample efficiencyAccuracy gain per additional annotation — the central measurement of this paper.

YOLOv8 Architecture Terms

Model
ComponentRole
BackboneFeature extractor — processes input image into spatial feature maps.
NeckFeature Pyramid Network — fuses multi-scale features for small-object detection.
HeadPredicts class probabilities and bounding-box coordinates.
Layer freezingLock backbone weights during early fine-tuning for stability at low N.
CheckpointSaved weights file (.pt) — used to resume, evaluate, or export the model.

Data Hygiene Checklist

Protocol
  • Keep strict train / val / test separation — no cross-contamination
  • Check for and remove duplicate scenes across splits (avoid leakage)
  • Keep annotation policy consistent across all crop types
  • Document split version and random seed for every experiment
  • Track class imbalance distribution before training starts

Suggested Learning Path

Onboarding
  1. 1
    Detection foundations IoU, precision, recall, F1, mAP — understand before touching model output
  2. 2
    Transfer learning Understand scratch vs pretrained vs fine-tune and when each makes sense
  3. 3
    YOLOv8 operation Architecture walkthrough + training controls (lr, epochs, freeze, imgsz)
  4. 4
    Experimental design Baseline ladder + fair low-label protocol + domain-split construction
  5. 5
    Research discipline Reproducible logs, session handoffs, and paper-ready experiment registry
Master all five blocks and you can execute the project, review the code, and defend the paper confidently.
