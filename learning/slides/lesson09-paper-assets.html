Lesson 09: Paper Assets

How GreenPoll turns experiment results into the arXiv table and figure.

File: scripts/build_paper_assets.py

First Principle: Papers Need Traceable Numbers

A result in a paper should be reproducible from stored experiment outputs, not copied from memory or hand-calculated in prose.

This repo uses results/summary.csv as the single source of truth for paper-facing metrics.

The Asset Pipeline

results/summary.csv
  -> scripts/build_paper_assets.py
  -> paper/arxiv/results_table.tex
  -> paper/arxiv/figures/sample_efficiency.pdf
  -> paper/arxiv/main.tex

The script exists to keep the table, figure, and logged results synchronized.

CSV Schema

result_id, run_id, method, heldout_crop, n_labels,
mAP50, mAP50_95, precision, recall, f1,
latency_ms, model_size_mb, train_time_min,
split_version, seed

These columns come from the baseline runner's SUMMARY_HEADER.

Aggregation

load_aggregated groups rows by:

(method, n_labels)

For each group, it computes:

  • Number of runs.
  • Mean mAP50.
  • Minimum and maximum mAP50.
  • Mean mAP50-95 when available.

Why Ignore Empty mAP50-95?

Some older rows have blank or effectively missing mAP50_95 values.

The script treats missing or zero-filled mAP50-95 values as absent, so it does not average fake zeros into newer runs.

Method Ordering

METHOD_ORDER = [
    "scratch",
    "coco_transfer",
    "crosspoll",
    "crosspoll_freeze10",
]

The table and figure use the same ordering and display labels, which keeps the story consistent across assets.

Budget Ordering

BUDGETS = [10, 25, 50, 100, 181]

The final budget is 181 because kiwi has 181 train images. This avoids presenting capped budgets as if they were distinct sample sizes.

Writing The Table

write_table creates a LaTeX table where each cell is:

mean mAP50 [min, max]

It also bolds the best mean per label budget.

The range communicates seed variability, which matters in few-shot experiments.

Writing The Figure

write_figure plots sample-efficiency curves:

  • X-axis: kiwi training labels, log-scaled.
  • Y-axis: test mAP@0.5.
  • Error bars: distance from mean to min/max over seeds.

The figure is saved to paper/arxiv/figures/sample_efficiency.pdf.

Headless Matplotlib

matplotlib.use("Agg")

Agg lets Matplotlib render files without opening a GUI window. That matters for CI, servers, and Kaggle-like environments.

Console Sanity Check

print_summary prints the numbers that will appear in the paper.

This is a human inspection step. It does not replace the generated table, but it helps catch surprising values before compiling the paper.

What Not To Do

Do not manually edit paper/arxiv/results_table.tex to fix a result. Fix or rerun the experiment, update results/summary.csv, then regenerate assets.

Connection To LaTeX

The arXiv paper lives under paper/arxiv. The generated table and figure are included from that manuscript tree.

The workflow is:

python scripts/build_paper_assets.py
compile paper/arxiv/main.tex

Key Takeaways

  • summary.csv is the numeric source of truth.
  • build_paper_assets.py performs aggregation, table generation, and figure generation.
  • Manual edits to generated assets break traceability.
  • Seed ranges should be read as uncertainty evidence, not decoration.
