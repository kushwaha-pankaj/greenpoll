Lesson 08: The Baseline Runner

The main sweep script that trains, evaluates, and logs the evidence table.

File: scripts/run_baselines.py

First Principle: A Fair Experiment Changes One Thing At A Time

A baseline runner should keep data splits, label budgets, metrics, and logging consistent while swapping only the method being compared.

Here, the method can be scratch, COCO transfer, or CrossPoll.

What This Script Does

  1. Subsample kiwi train images to a label budget.
  2. Write a temporary YOLO data.yaml.
  3. Train YOLOv8n from the selected initialization.
  4. Evaluate the best checkpoint on kiwi test.
  5. Append metrics to results/summary.csv and the run registry.

Method Comparison

Initializations

  • scratch: architecture only, random weights.
  • coco_transfer: COCO-pretrained yolov8n.pt.
  • crosspoll: source-crop flower checkpoint.

Constant Target

All methods adapt to held-out kiwi with the same budgets and seed set.

Config Loading

At import time, the script reads configs/experiments/crosspoll.yaml.

DEFAULT_SEEDS = CFG["reproducibility"]["seeds"]
DEFAULT_BUDGETS = CFG["data"]["label_budgets"]
EPOCHS = CFG["training"]["epochs"]
LR_FINETUNE = CFG["training"]["lr0_finetune"]

This keeps the command-line defaults aligned with the experiment protocol.

Device Selection

GREENPOLL_DEVICE
  -> YAML device
  -> MPS if available
  -> CUDA if YAML says MPS but host has CUDA
  -> Ultralytics auto-detect

This exists because local Mac development and Kaggle GPU execution need different device strings.

Listing Image/Label Pairs

for img in sorted(img_dir.iterdir()):
    if img.suffix.lower() in (".jpg", ".jpeg", ".png"):
        lbl = lbl_dir / (img.stem + ".txt")
        if lbl.exists():
            pairs.append((img, lbl))

The script only trains on images that have corresponding label files.

Few-Shot Budgeting

rng = random.Random(seed)
n = min(budget, len(pairs))
selected = rng.sample(pairs, n)

The "label budget" in this script is a number of kiwi training images selected by seed.

Because kiwi train has 181 images, budgets above 181 are capped. That prevents duplicated effective budgets.

Temporary Dataset

The subsampled kiwi train set is created under runs/tmp/<run_id>.

Images and labels are symlinked for speed and disk efficiency, then a temporary data.yaml points Ultralytics at:

train: runs/tmp/.../train/images
val: data/kiwi/valid/images
test: data/kiwi/test/images

Why Absolute Paths In data.yaml?

Training frameworks often run from changing working directories. Absolute paths remove ambiguity about where images and labels live.

The script writes stringified Path values into the generated YAML.

Run IDs

method_tag = f"{method}_freeze{freeze}" if freeze > 0 else method
run_id = f"{method_tag}_kiwi_b{budget}_s{seed}"

The run ID encodes method, crop, budget, and seed. Freeze variants get distinct IDs so they do not overwrite the original CrossPoll rows.

Dedupe Logic

load_completed_run_ids reads existing summary.csv rows and skips matching run_ids.

This makes interrupted sweeps resumable. It also means changing a run's semantics without changing the run ID is dangerous.

Creating The Model

scratch       -> YOLO("yolov8n.yaml")
coco_transfer -> YOLO("yolov8n.pt")
crosspoll     -> YOLO("runs/joint/.../best.pt")
              or YOLO("checkpoints/joint_pretrain.pt")

The three branches define the central comparison in the paper.

Training Call

The script passes a consistent set of training keyword arguments to Ultralytics:

  • Data, epochs, patience, image size, batch, optimizer, AMP.
  • Fine-tuning learning rate, cache, cosine LR, workers, deterministic flag.
  • Seed, project directory, run name, optional device, optional freeze.

Freeze And Patience Overrides

The CLI includes --freeze, --patience-override, and --lr0-override.

These exist because early CrossPoll fine-tuning can forget source-crop features at small budgets. Freezing early layers can protect low-level visual features.

Evaluation Metrics

metrics = eval_model.val(data=str(data_yaml), split="test")
mAP50 = float(metrics.box.map50)
mAP50_95 = float(metrics.box.map)
precision = float(metrics.box.mp)
recall = float(metrics.box.mr)
f1 = 2 * precision * recall / (precision + recall)

The final row logs both detection quality and operational metadata.

Logging Contracts

  • append_results writes paper-facing metrics to results/summary.csv.
  • append_registry writes run provenance to experiments/registry.csv.
  • SUMMARY_HEADER defines column order when creating a fresh summary file.

Cleanup

After a real run, the temporary dataset is removed with shutil.rmtree(tmp_dir).

That keeps only the meaningful artifacts: run outputs, checkpoint paths, registry rows, and summary metrics.

CLI Surface

python scripts/run_baselines.py
python scripts/run_baselines.py --seeds 42 --budgets 25
python scripts/run_baselines.py --methods crosspoll --freeze 10 --patience-override 30

The same script supports smoke tests, full sweeps, and targeted CrossPoll variants.

Key Takeaways

  • run_baselines.py is the main evidence-producing script.
  • A label budget is implemented as sampled kiwi training images.
  • Run IDs are the dedupe key and must reflect meaningful protocol changes.
  • All paper-facing metrics should flow through results/summary.csv.
