Environment Setup & Project Verification

Make sure everything runs before writing a single line of experiment code.
Why this step first?

A broken environment wastes hours of research time. Before touching data or models we need a clean, reproducible Python environment where all dependencies install cleanly, the package discovers its own source tree, and the existing unit-tests pass green. This step costs ~20 minutes and saves days of "it worked on my machine" problems later.

What you need before starting
  • Python 3.11+ installed (check: python --version)
  • Git repository already cloned (you're reading this, so ✓)
  • A terminal open inside the repo root folder
  • ~2 GB free disk space for the virtual environment + Ultralytics
Task 1 — Create & activate the virtual environment

We use a .venv folder (already present from your last session). If it does not exist, create it first. 

# If .venv doesn't exist yet:
python -m venv .venv

# Activate (macOS / Linux):
source .venv/bin/activate

# Confirm you're inside the venv:
which python        # should show …/greenpoll/.venv/bin/python
Task 2 — Install the project in editable mode

The pyproject.toml defines all dependencies. Installing with -e .[dev] makes the greenpoll package importable from anywhere and installs extra dev tools (pytest, etc.). 

pip install -e ".[dev]"

This pulls in Ultralytics YOLOv8, PyTorch, and testing libraries. It may take 3–5 minutes the first time. You should see no red ERROR lines at the end.

Task 3 — Run the existing unit-tests

Two test files already exist. They cover the prioritize.scoring module and the plan.baselines nearest-neighbour route planner. All tests must pass before we go any further. 

pytest tests/ -v

Expected output: 2 test files, all green, 0 failures.
If a test fails, fix it now — these are the foundation.

Task 4 — Verify the CLI entry-point exists
greenpoll --help

Even if the CLI does nothing yet, the command must resolve without a executable not found error. This confirms the package is properly installed.

Task 5 — Confirm the experiment config loads

The locked experiment config lives at configs/experiments/crosspoll.yaml. Quickly verify it is valid YAML and all required keys are present. 

python -c "
import yaml, pathlib
cfg = yaml.safe_load(pathlib.Path('configs/experiments/crosspoll.yaml').read_text())
print('Project :', cfg['project']['name'])
print('Heldout  :', cfg['data']['heldout_crop'])
print('Budgets  :', cfg['data']['label_budgets'])
print('OK — config valid')
"
Definition of Done for Step 1
  • pip install -e .[dev] completes with no errors
  • pytest tests/ -v — 0 failures
  • greenpoll --help — resolves without error
  • Config script prints "OK — config valid" with correct values
